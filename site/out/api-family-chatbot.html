API Family: Chatbot

Overview

The Chatbot API family provides endpoints for health checks, chatbot query processing, and user/report management. These endpoints are used by frontend clients, monitoring tools, and integration services to interact with the DeepDoc chatbot backend, verify service health, submit developer questions, and manage user/report resources.

The main endpoints are:

  • Health checks: Verify backend readiness (GET /health, POST /health)
  • Chatbot queries: Submit developer questions and receive grounded answers (POST /query)
  • Report retrieval: Fetch report details by slug (GET /reports/{slug})
  • User management: List, create, retrieve, delete users, and fetch user stats (GET/POST/DELETE /api/users, GET /api/users/{id}/stats)
These endpoints are served both via FastAPI (see deepdoc/chatbot/service.py:528-545) and Django REST Framework (see tests/fixtures/frameworks/django_app/urls.py:9-43). The FastAPI implementation is scaffolded for chatbot-enabled repos (Integration: Deepdoc Chat API).

Authentication

FastAPI Endpoints (deepdoc/chatbot/service.py:528-545)

  • No authentication required for /health and /query endpoints.
  • CORS middleware is applied via CORSMiddleware in create_fastapi_app() (deepdoc/chatbot/service.py:513), allowing origins from chatbot_allowed_origins() (deepdoc/chatbot/settings.py:126).
  • Credentials are not required; all endpoints are public unless restricted by deployment config.

Django Endpoints (tests/fixtures/frameworks/django_app/urls.py:9-43)

  • No authentication required for /health and /reports/{slug}.
  • User endpoints (/api/users*) are routed via UserViewSet (ModelViewSet), which may enforce authentication depending on DRF settings (not visible in evidence).
If deploying in production, review CORS and authentication settings in deepdoc/chatbot/settings.py and your Django/DRF config. See Setup & Getting Started for environment setup.

Execution Flow

Chatbot Query (POST /query)

  1. Request received by FastAPI handler (deepdoc/chatbot/service.py:532).
  2. Request body parsed as QueryRequest (deepdoc/chatbot/service.py:18).
  3. Service instantiated: ChatbotQueryService (deepdoc/chatbot/service.py:25) loads corpus records, vector indices, and config.
  4. Query expansion: _expand_query() (deepdoc/chatbot/service.py:164) generates alternative queries via LLM if retrieval_cfg["query_expansion"] is enabled.
  5. Embedding: embedding_client.embed() (deepdoc/chatbot/providers.py:136) embeds all query variants.
  6. Similarity search: _multi_query_search() (deepdoc/chatbot/service.py:186) searches code, artifact, doc, and relationship corpora, merging results.
  7. Chain retrieval: _chain_retrieve() (deepdoc/chatbot/service.py:217) pulls in code chunks from files referenced in relationship hits.
  8. Reranking: _rerank() (deepdoc/chatbot/service.py:260) uses LLM to rerank chunks if retrieval_cfg["rerank"] is enabled.
  9. Prompt assembly: _build_prompt() (deepdoc/chatbot/service.py:402) constructs the answer context.
  10. Response returned: Answer, citations, links, and chunk count.

Conditional Branches & Feature Flags

  • retrieval_cfg["query_expansion"]: Enables LLM query expansion.
  • retrieval_cfg["rerank"]: Enables LLM-based reranking.
  • retrieval_cfg["top_k_code"], top_k_artifact, top_k_docs, top_k_relationship: Limits for search results.
  • retrieval_cfg["max_prompt_code_chunks"], max_prompt_artifact_chunks: Prompt chunk limits.
  • If no context found: _no_context_result() (deepdoc/chatbot/service.py:150) returns a fallback answer.

Error Handling

  • Any exception in query() returns {"error": "chatbot_query_failed", "detail": ...} with HTTP 500.
  • Startup errors in scaffolded backend return {"error": "startup_failed", "detail": ...} with HTTP 503.

State Changes & Side Effects

  • No database writes: All chatbot endpoints are read-only.
  • Corpus index access: Loads code, artifact, doc, and relationship records via load_corpus() (deepdoc/chatbot/persistence.py:34).
  • Vector search: Uses load_vector_index() (deepdoc/chatbot/persistence.py:58) and similarity_search() (deepdoc/chatbot/persistence.py:131).
  • LLM calls: Uses build_chat_client() and build_embedding_client() (deepdoc/chatbot/providers.py:132,136).
  • No background jobs, cache invalidations, or webhooks are triggered by these endpoints.
