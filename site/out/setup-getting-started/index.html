Setup & Getting Started

Welcome to DeepDoc! This guide will walk you through installing, configuring, and running DeepDoc in your project. It covers the core setup workflow, configuration files, environment variables, and common developer patterns. For a high-level system overview, see DeepDoc Architecture & Overview.

Overview

DeepDoc is an AI-powered documentation generator for codebases. It scans your repository, plans documentation buckets, generates MDX pages (with Mermaid diagrams), and builds a Fumadocs-powered site. The setup process ensures DeepDoc is correctly installed, configured, and ready to generate docs for your project.

  • Installation: Via PyPI or from source.
  • Configuration: Managed via .deepdoc.yaml and environment variables.
  • CLI Usage: Initialize, generate, update, and serve documentation.
  • Chatbot Support: Optional, with extra dependencies and config.

For details on the full architecture and pipeline, see DeepDoc Architecture & Overview.

How It Works

The setup and initialization flow for DeepDoc involves several key steps and components:

  • Install: Get DeepDoc via PyPI or from source.
  • Initialize: deepdoc init creates .deepdoc.yaml in your repo.
  • Configure: Set required API keys as environment variables.
  • Generate: deepdoc generate scans, plans, and generates docs.
  • Preview: deepdoc serve launches the local Fumadocs site.
  • Chatbot (optional): Install extras, configure, and run chatbot services.

For CLI command details, see CLI Commands & Usage.

Configuration

1. .deepdoc.yaml

All project-level configuration is stored in .deepdoc.yaml at your repo root. This file is created by deepdoc init and can be edited manually.

Key fields:

  • project_name, description
  • output_dir, site_dir
  • llm (provider, model, API key env)
  • chatbot (see below)
  • include, exclude (glob patterns)
  • github_pages, site (branding)

Gotchas & Edge Cases

If you run deepdoc generate in a directory with existing docs not managed by DeepDoc, it will refuse to overwrite them unless you use --force or --clean --yes.

  • API Key Resolution: If the environment variable for your LLM or chatbot provider is not set, DeepDoc will fail to authenticate. The variable name is resolved from config (api_key_env field).
  • Chatbot CORS Origins: If you set DEEPDOC_CHATBOT_PREVIEW_PORT, chatbot_allowed_origins() (deepdoc/chatbot/settings.py:126) will append http://localhost:<port> and http://127.0.0.1:<port> to allowed origins. This is critical for local frontend preview.
  • Config Merging: Both deepdoc/config.py:_deep_merge() and deepdoc/chatbot/settings.py:_deep_merge() perform recursive merges. If you have nested dicts in your .deepdoc.yaml, user values override defaults deeply.
  • Port Calculation: If no backend URL is set for the chatbot, chatbot_backend_port() (deepdoc/chatbot/settings.py:116) will deterministically assign a port based on your repo path using CRC32. This avoids port conflicts for multiple checkouts.
  • Missing .deepdoc.yaml: If you forget to run deepdoc init, load_config() will fall back to defaults, but you won't have project-specific settings.
