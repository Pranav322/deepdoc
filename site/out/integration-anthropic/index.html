Integration: Anthropic

What This Integration Does

The Anthropic integration enables DeepDoc to leverage Anthropic's Claude LLM models for chat and completion tasks. This integration is responsible for generating natural language documentation, clustering code features, and performing business-domain analysis via LLM completions. Anthropic is the default LLM provider for DeepDoc, powering core documentation generation and code analysis workflows.

Anthropic is used as the primary LLM backend for generating documentation, clustering features, and other advanced language tasks in DeepDoc.

Where It Enters the Codebase

The Anthropic integration is woven into DeepDoc's LLM abstraction layer and configuration system. The main entry points are:

  • Client Wrapper:
    • LLMClient (deepdoc/llm/client.py:12):
      A thin wrapper around LiteLLM, which supports Anthropic as a backend. Handles request construction, authentication, and streaming.
  • Config Management:
    • DEFAULT_CONFIG (deepdoc/config.py:11):
      Sets Anthropic as the default provider, model, and API key environment variable.
    • resolve_api_key() (deepdoc/config.py:230):
      Looks up the Anthropic API key from environment variables.
  • CLI Initialization:
    • init command (deepdoc/cli.py:131):
      When initializing a new DeepDoc project, Anthropic is the default LLM provider. The CLI scaffolds .deepdoc.yaml with Anthropic-specific settings.
  • LLM Usage in Feature Extraction:
    • cluster_giant_file() (deepdoc/scan_v2.py:65):
      Uses LLMClient to send clustering prompts to Anthropic.
    • discover_integrations() (deepdoc/scan_v2.py:572):
      Optionally uses Anthropic (via LLMClient) to normalize integration signals.

Participating Features & Endpoints

Anthropic is used by all DeepDoc features that require LLM completions, including:

  • Documentation Generation:
    • Pipeline: Document Generation & Planning:
      Uses Anthropic to generate page content, summaries, and feature clusters.
  • Feature Clustering:
    • Parsing & Source Analysis:
      Sends symbol inventories to Anthropic for business-domain grouping.
  • Integration Discovery:
    • Integration: Vinculum Warehouse Management:
      LLM normalization of integration signals may use Anthropic.
  • CLI Workflows:
    • CLI Commands & Usage:
      All commands that trigger documentation generation (generate, update) will use Anthropic if configured.

Anthropic is the default LLM for all core DeepDoc workflows unless explicitly changed in .deepdoc.yaml.

Request/Response or Message Flow

The interaction with Anthropic is mediated by the LLMClient (deepdoc/llm/client.py:12), which uses LiteLLM to send chat completion requests. The flow is as follows:

Request Construction

  • Endpoint:
    • Anthropic's /v1/messages (via LiteLLM)
  • Headers:
    • x-api-key: Value from ANTHROPIC_API_KEY
  • Payload:
    • model: e.g., claude-3-5-sonnet-20241022
    • messages:
      • {"role": "system", "content": ...}
      • {"role": "user", "content": ...}
    • temperature: From config (default: 0.2)
    • max_tokens: Optional, from config
    • base_url: Optional, for custom endpoints

Retry, Reconciliation & Failure Handling

Error Handling

  • Import Errors:
    • If LiteLLM is not installed, LLMClient.complete() (deepdoc/llm/client.py:30) raises a RuntimeError with a clear message.
  • Request Failures:
    • Any exception during the LLM request is caught and re-raised as RuntimeError(f"LLM request failed: {e}").
  • API Key Missing:
    • If ANTHROPIC_API_KEY is not set, requests will fail with an authentication error from Anthropic.

No automatic retry or circuit breaker logic is implemented in the LLM client layer.
Retries, if needed, must be handled at a higher level or by wrapping calls to LLMClient.

If the Anthropic API is down or the API key is invalid, documentation generation will fail with a fatal error. There is no built-in retry or fallback to another provider unless you reconfigure .deepdoc.yaml.

Operational Gotchas

  • Rate Limits:
    • Anthropic enforces per-key and per-organization rate limits. Exceeding these will result in HTTP 429 errors.
  • Timeouts:
    • No explicit timeout is set in LLMClient; timeouts are determined by LiteLLM or Anthropic defaults.
  • Payload Size:
    • Large prompt payloads (e.g., giant file clustering) may hit context window limits. The max_tokens and prompt construction are controlled via config.
  • Versioning:
    • The default model is claude-3-5-sonnet-20241022. Model upgrades may change output style or capabilities.
  • Environment Consistency:
    • The API key must be present in the environment for all CLI and programmatic invocations. The .env file is loaded automatically by DeepDoc's CLI (_autoload_repo_env() in deepdoc/cli.py:95).

Use the deepdoc config show and deepdoc config set commands to inspect and modify Anthropic-related settings. See CLI Commands & Usage.
