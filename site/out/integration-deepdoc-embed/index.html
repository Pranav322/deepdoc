Integration: Deepdoc Embed API

What This Integration Does

The Deepdoc Embed API provides vector embedding services for DeepDoc's code and artifact retrieval features. It is responsible for transforming code snippets, documentation, and other textual artifacts into vector representations, enabling semantic search, similarity matching, and context retrieval for the DeepDoc Chatbot and documentation generation pipeline.

This integration is essential for:

  • Powering code and artifact search within the API Family: Chatbot.
  • Supporting advanced retrieval workflows, such as query expansion and reranking, to deliver relevant context for user queries.
  • Enabling similarity search over indexed code, documentation, and artifacts.

The Deepdoc Embed API is a required dependency for all chatbot and retrieval features. If not configured, code search and context-aware answers will not function.

Where It Enters the Codebase

The Deepdoc Embed API is referenced and used in several key locations:

  • Backend Scaffold:
    • deepdoc/chatbot/scaffold.py
      • The .env.example scaffold includes DEEPDOC_EMBED_API_KEY=, indicating the need for an API key.
      • The generated settings.py sets the base URL for the chatbot backend, which will use the Embed API for vectorization.
  • Chatbot Settings & Config:
    • deepdoc/chatbot/settings.py
      • Embedding provider configuration is defined under the embeddings key in DEFAULT_CHATBOT_CONFIG.
      • The API key is referenced via the api_key_env field (default: DEEPDOC_EMBED_API_KEY).
  • CLI Integration & Environment Handling:
    • deepdoc/cli.py
      • The CLI loads .env files and ensures the embedding API key is available in the environment.
      • The init command writes the correct config and scaffolds the chatbot backend with embedding support.
  • Config Management:
    • deepdoc/config.py
      • Loads and merges the embedding provider configuration from .deepdoc.yaml.
  • Integration Detection:
    • deepdoc/scan_v2.py
      • Integration signals for the Embed API are detected via environment variables and SDK usage patterns.

Key config and environment variables:

  • DEEPDOC_EMBED_API_KEY (used throughout for authentication)
  • Embedding provider settings in .deepdoc.yaml under chatbot.embeddings

Participating Features & Endpoints

The Deepdoc Embed API is used by the following features and endpoints:

  • Chatbot Query Endpoint
    See API Family: Chatbot for details on /query and /health endpoints.
  • Code and Artifact Retrieval
    All code and artifact search features rely on embedding vectors produced by this integration.
  • Frontend UI Integration
    The Frontend & UI Integration Overview and Frontend Documentation Site Builder depend on the backend's ability to retrieve relevant code and documentation chunks using embeddings.
  • Pipeline Generation
    The Pipeline: Document Generation & Planning leverages embedding-based retrieval for evidence assembly.

Auth & Configuration

Authentication:
The Deepdoc Embed API requires an API key, provided via the DEEPDOC_EMBED_API_KEY environment variable.

Configuration locations:

  • Environment Variable:

      Must be set in your .env file or environment before starting the backend.

  • Config File (.deepdoc.yaml):

How to set up:

Obtain an Embed API Key Request or generate a valid Deepdoc Embed API key for your organization.

Configure Environment Add the following to your .env file at the repo root:


DEEPDOC_EMBED_API_KEY=your-embed-api-key

Verify Configuration Ensure your .deepdoc.yaml contains the correct chatbot.embeddings section. See Setup & Getting Started for full environment setup.

Restart the Backend Restart the chatbot backend so it picks up the new environment variable.

All three environment variables must be set before starting: DEEPDOC_EMBED_API_KEY, DEEPDOC_CHAT_API_KEY, and any LLM provider key (e.g., ANTHROPIC_API_KEY). See Integration: Anthropic for LLM config.

Retry, Reconciliation & Failure Handling

Error Handling:

  • If the Embed API is unreachable or the API key is missing/invalid, embedding requests will fail.
  • In tests/test_chatbot_query.py, the _FailingEmbedClient simulates a missing API key: The backend responds with a 500 error and a JSON payload:
  • The /query endpoint will return a 503 or 500 error if embedding fails during startup or at runtime.

No Retry Logic by Default:

  • There is no built-in retry or circuit breaker in the backend scaffold (deepdoc/chatbot/scaffold.py:30).
  • If embedding fails, the error is surfaced to the user via the API response.

Fallback Behavior:

  • If the backend cannot start due to missing dependencies, it exposes a /health endpoint that returns a clear error message.
  • If embedding fails during a query, the chatbot returns an error response and does not attempt to answer with stale or partial data.

There is no automatic retry for failed embedding requests. Ensure the Embed API is highly available and monitor for failures.

Operational Gotchas

  • API Key Required:
    The DEEPDOC_EMBED_API_KEY must be present in the environment. If missing, all retrieval and chatbot features will fail.
  • Batch Size:
    The default embedding batch size is 1, but can be increased via chatbot.embeddings.batch_size in .deepdoc.yaml for better throughput.
  • Timeouts & Rate Limits:
    The backend does not implement custom timeouts or rate limiting for embedding requests. If the Embed API enforces limits, you must handle them at the infrastructure level.
  • Payload Size:
    Large batches or very long texts may exceed the Embed API's payload limits. Tune batch_size and chunking settings as needed.
  • Versioning:
    The embedding model and API version can be set in config (model, api_version). Ensure compatibility with your Embed API deployment.
  • Monitoring:
    Monitor for 500/503 errors on the /query endpoint and backend logs for embedding failures.

Use batch processing for large datasets — it's 10x faster. Adjust batch_size in .deepdoc.yaml as needed.
