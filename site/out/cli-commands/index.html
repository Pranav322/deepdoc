CLI Commands & Usage

Overview

The DeepDoc CLI (deepdoc/cli.py) is the primary interface for generating, updating, previewing, and deploying documentation for your codebase. It orchestrates the full documentation lifecycle, from initializing project config to serving a live preview and deploying a static site. The CLI is tightly integrated with DeepDoc's planning, generation, and site-building pipeline, and interacts with LLM providers (Anthropic, OpenAI, Azure), chatbot APIs (Deepdoc Chat API), and embedding services (Deepdoc Embed API).

The CLI is your entry point for all DeepDoc workflows. For architectural context, see DeepDoc Architecture & Overview.

How It Works

The CLI exposes a set of commands for managing documentation. Typical workflow:

Initialize the project Run deepdoc init to create .deepdoc.yaml and set up config.

Generate documentation Run deepdoc generate to scan the codebase, plan docs, and write pages.

Preview locally Run deepdoc serve to launch a live-reloading Fumadocs site.

Update docs after changes Run deepdoc update for incremental refresh after code changes.

Deploy static site Run deepdoc deploy to build and publish the static docs.

Gotchas & Edge Cases

  • Output Directory State: If docs/ exists but is not DeepDoc-managed, generate will refuse to overwrite unless --clean --yes is used. This prevents accidental deletion of non-DeepDoc docs.
  • Config Overwrite: init prompts before overwriting .deepdoc.yaml. If you skip, initialization aborts.
  • Node.js Requirement: serve and deploy require Node.js >= 18. If missing, CLI exits with error.
  • Chatbot Backend: If chatbot is enabled but backend port is busy, CLI will pick the next available port and warn.
  • Environment Variables: Missing LLM API keys or misconfigured .env will cause failures at generation or preview.
  • Clean Command: Irreversibly deletes DeepDoc artifacts and config. Use with caution.
  • Batch Size and Parallelism: Large batch sizes or high parallel workers can hit rate limits on LLM providers. Use --rate-limit-pause to throttle.
Running deepdoc clean will permanently delete all DeepDoc output and state. Confirm before proceeding.
Use deepdoc generate --batch-size and --max-parallel-workers for large repositories, but monitor LLM rate limits.
