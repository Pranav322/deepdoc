Frontend & UI Integration Overview

Overview

The Frontend & UI Integration system component orchestrates how documentation content, code, and artifact summaries are extracted, chunked, and prepared for rendering in the DeepDoc documentation frontend. It acts as the bridge between backend knowledge extraction and the site-building pipeline, ensuring that both code and documentation are processed into retrievable, navigable units for the UI.

This component is critical for enabling features such as:

  • Searchable, chunked documentation and code in the frontend
  • Linking between code, artifacts, and documentation pages
  • Trust scoring and tiering for content surfaced in the UI

It integrates tightly with the Frontend Documentation Site Builder, and is foundational to the overall DeepDoc Architecture & Overview.

For details on how Vue components are parsed and surfaced in the frontend, see Vue Component Parsing and Integration.

Architecture & Design

The Frontend & UI Integration layer sits between the backend planning/parsing pipeline and the frontend site builder. It receives parsed code, documentation, and artifact data, then processes this into standardized "chunks" (see ChunkRecord in deepdoc/chatbot/types.py:13) for retrieval and rendering.

Key design patterns:

  • Chunking: All content (code, docs, artifacts) is split into manageable, retrievable units with metadata for UI navigation and search.
  • Trust & Tiering: Each chunk is assigned a trust score and publication tier, influencing its prominence in the frontend.
  • Relationship Mapping: Chunks are linked to their owning documentation buckets/pages for cross-navigation.

The chunking and summary extraction logic is decoupled from the frontend rendering, enabling flexible UI integration and future extensibility.

Configuration

All chunking and summary extraction behavior is controlled by the chatbot config, merged via get_chatbot_cfg() (deepdoc/chatbot/settings.py:77).
Key config knobs:

Config KeyDescription
chunking.code_chunk_linesMax lines per code chunk
chunking.code_chunk_overlapOverlap lines between code chunks
chunking.artifact_chunk_linesMax lines per artifact chunk
chunking.artifact_chunk_overlapOverlap lines between artifact chunks
chunking.max_doc_summary_chunks_per_pageMax summary chunks per doc page
chunking.max_doc_summary_charsMax chars per summary chunk

All chunking parameters can be overridden in the chatbot config dictionary passed to these functions.

How Other Components Use This

  • The Frontend Documentation Site Builder consumes the output of build_code_chunks(), build_artifact_chunks(), and build_doc_summary_chunks() to assemble the static documentation site.
  • The Pipeline: Document Generation & Planning orchestrates when and how these chunking functions are called.
  • The Vue Component Parsing and Integration page details how Vue-specific chunks are surfaced in the frontend.
  • Integration: Deepdoc Chat API and Integration: Deepdoc Embed API rely on the chunked data for retrieval and embedding.

Edge Cases & Failure Modes

  • Missing Files: If a file referenced in the plan does not exist, chunking functions skip it silently.
  • Giant Files: Files detected as "giant" are chunked using special cluster logic to avoid memory issues.
  • Malformed Frontmatter: _strip_frontmatter() in deepdoc/chatbot/docs_summary.py:95 is robust to malformed or missing frontmatter, falling back to raw content.
  • Overlapping Chunks: _deduplicate_overlapping() in deepdoc/chatbot/chunker.py:544 removes near-duplicate chunks with >50% line overlap.
  • Truncation: If a chunk's text exceeds MAX_CHUNK_CHARS (6000), it is truncated with a marker to fit embedding model limits.
  • Legacy Plan Compatibility: Uses both legacy and current DocPlan/DocPage types for backward compatibility.

If chunking config is set too aggressively (e.g., very small chunk sizes), frontend navigation and search quality may degrade.
