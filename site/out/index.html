DeepDoc Architecture & Overview

Auto-generate deep engineering documentation from real codebases using AI.

What This Does

DeepDoc is an AI-powered documentation engine that scans your codebase, analyzes its structure, and generates a fully navigable, developer-focused documentation site. It serves engineering teams who need accurate, maintainable docs for complex, multi-language projects. DeepDoc covers everything from system architecture to endpoint references, integrations, and operational guides—grounded in real source code.

Architecture Overview

At a high level, DeepDoc operates as a multi-phase pipeline that ingests your repository, plans a documentation structure, generates rich MDX pages (with diagrams), and builds a local-first Fumadocs site. The architecture is modular, with clear separation between scanning, planning, generation, and frontend site building.

  • Repo Scan: Walks the repo, parses supported languages, detects endpoints, config, and integrations.
  • Doc Planning: Classifies files, proposes documentation "buckets" (system, feature, endpoint, integration, etc.), and assigns files/symbols.
  • Evidence Assembly: Gathers source, config, endpoint, and integration evidence for each bucket.
  • Page Generation: Uses LLMs to generate validated MDX pages, including diagrams and cross-links.
  • Fumadocs Site Builder: Assembles the documentation site as a Next.js app with Fumadocs UI.
  • OpenAPI Import: Detects and imports OpenAPI/Swagger specs for canonical API docs.
  • Manifest & Incremental Update: Tracks file hashes for efficient updates.

Every major component is documented in detail—see the Pipeline: Document Generation & Planning and Frontend Documentation Site Builder pages for deep dives.

End-to-End Runtime Flow

The main flow for generating documentation is as follows:

Key files and their roles:

  • deepdoc/planner_v2.py: Scans and plans the documentation structure.
  • deepdoc/generator_v2.py: Assembles evidence and generates validated documentation pages.
  • deepdoc/site/fumadocs_builder_v2.py: Builds the Fumadocs-based frontend site.
  • deepdoc/openapi.py: Handles OpenAPI/Swagger spec import for API docs.
  • README.md: Provides onboarding and setup context.

For a detailed breakdown of each pipeline phase, see Pipeline: Document Generation & Planning.

Tech Stack

  • Languages: Python, JavaScript, TypeScript, Go, PHP, Vue
  • Frameworks Detected: Django, Falcon, FastAPI, Express, Fastify, Laravel, Vue, Next.js, Nuxt, Fiber, Gin, Flask, NestJS
  • Frontend: Next.js + Fumadocs UI (MDX, Mermaid, Orama search)
  • LLM Providers: Anthropic, OpenAI, Azure OpenAI, Ollama (configurable)
  • Storage: No database required for docs generation; uses local file system and manifest tracking.
  • CI/CD: GitHub Actions workflows for release and test automation.
  • Deployment: Static export via Next.js (site/out/), suitable for any static host.

The detected frameworks shape how DeepDoc parses, analyzes, and documents your codebase. See Setup & Getting Started for environment and dependency details.

Project Structure

  • deepdoc/: Core engine (planning, generation, site building, prompts, OpenAPI import)
  • deepdoc/parser/: Language and framework-specific parsers
  • deepdoc/site/: Fumadocs site builder and React/TSX components
  • tests/: Framework detection, planner, and builder test fixtures

See Frontend Documentation Site Builder for details on the site generation process.

Data, State, And Integrations

  • Key Models & State: No persistent DB for docs; state is tracked via manifest files (deepdoc/manifest.py).
  • Caches & Queues: Not used in core pipeline; background jobs are handled via CLI commands.
  • Background Workers: Not required for docs generation; all processing is batch/CLI-driven.
  • Integrations:
    • OpenAPI/Swagger: Imported for canonical API docs (API Family: Chatbot)
    • Vinculum Warehouse Management: Detected and documented as an integration (Integration: Vinculum Warehouse Management)
    • Webhook Handlers: Supported and documented (Integration: Webhook Handlers)
    • Anthropic, OpenAI, Ollama: LLM providers for doc generation (Integration: Anthropic)

All integrations are detected and documented based on actual code evidence. See the Integrations section for details.

Key Concepts And Gotchas

Getting Started

Install DeepDoc Run pip install deepdoc (or pip install "deepdoc[chatbot]" for chatbot features).

Initialize in your repo Run deepdoc init in your project root to create a .deepdoc.yaml config.

Set your LLM API key Export your API key, e.g. export ANTHROPIC_API_KEY=sk-ant-....

Generate documentation Run deepdoc generate to scan, plan, and generate docs.

Preview the docs site Run deepdoc serve and open http://localhost:3000.

See Setup & Getting Started for full installation, environment, and troubleshooting details.
