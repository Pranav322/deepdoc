Development Notes

Overview

This page summarizes the repository-level development notes and design context for DeepDoc, as captured in README.md. It covers the architectural principles, key features, installation and setup workflows, and configuration conventions that underpin the DeepDoc system. Understanding these notes is essential for new developers onboarding to DeepDoc, as they clarify the rationale behind bucket-based documentation, the five-phase pipeline, and the integration of AI-driven planning and generation. This context also provides practical guidance for installation, configuration, and typical usage patterns, directly informing how you interact with the CLI, configure LLM providers, and manage documentation updates.

This page is a high-level context reference. For detailed architectural flows and runtime service internals, see DeepDoc Architecture & Overview and Runtime Services & Operations.

Key Findings or Terms

  • Bucket-Based Documentation Architecture
    DeepDoc organizes documentation into buckets: system, feature, endpoint, endpoint reference, integration, and database. This avoids the pitfalls of one-file-per-page output and enables logical grouping of related artifacts.
    Source: README.md: Features

  • Five-Phase Pipeline
    The documentation generation process is split into five phases:

    1. Scan (repo walk and artifact detection)
    2. Plan (multi-step AI bucket planner)
    3. Generate (parallel page generation)
    4. API Ref (OpenAPI asset staging)
    5. Build (site generation with Fumadocs and Orama search)
      Source: README.md: Features, README.md: deepdoc generate

  • Multi-Language and Framework Support
    DeepDoc parses Python, JavaScript, TypeScript, Go, PHP, and Vue, with high-confidence support for FastAPI, Flask, Laravel, Django/DRF, Express, Fastify, and Vue SFC.
    Source: README.md: Supported Languages & Frameworks

  • Incremental Updates and Safe Existing-Docs Behavior
    The deepdoc update command uses persisted plan and ledger data to regenerate only stale or structurally affected docs. Plain generate refuses to overwrite non-DeepDoc folders unless explicitly cleaned.
    Source: README.md: Features, README.md: deepdoc generate, README.md: deepdoc update

  • Configurable LLM Providers
    DeepDoc supports Anthropic, OpenAI, Azure OpenAI, Ollama, and any LiteLLM-compatible provider. Configuration is managed via .deepdoc.yaml and environment variables.
    Source: README.md: LLM Provider Setup, README.md: Configuration

  • Chatbot Integration
    DeepDoc can scaffold an AI-powered chatbot backend using FastAPI, FAISS, and Azure/OpenAI embedding models. Chatbot configuration is independent from the main LLM settings and is managed under the chatbot key in .deepdoc.yaml.
    Source: README.md: Chatbot

  • Generated Artifacts and Site Structure
    After generation, DeepDoc produces .deepdoc/ state, bucket plans, scan caches, and a Fumadocs-powered site in site/.
    Source: README.md: Generated Files

  • CI/CD and Release Automation
    DeepDoc provides GitHub Actions workflows for automated doc updates and PyPI releases, with changelog-driven release notes.
    Source: README.md: GitHub Actions CI/CD, README.md: Releasing

Relevance to the Codebase

The development notes in README.md directly inform the following areas:

  • Architecture & Planning
    The bucket-based design and five-phase pipeline are implemented in deepdoc/planner_v2.py, deepdoc/generator_v2.py, and deepdoc/pipeline_v2.py. See Pipeline: Document Generation & Planning for details.

  • Runtime Services
    Chatbot backend and vector indexing are handled in deepdoc/chatbot/service.py, deepdoc/chatbot/providers.py, and deepdoc/chatbot/indexer.py. See Runtime Services & Operations.

  • Parsing & Source Analysis
    Multi-language parsing logic is found in deepdoc/parser/__init__.py, deepdoc/parser/vue_parser.py, and related files. See Parsing & Source Analysis.

  • Frontend & Site Generation
    The Fumadocs site builder logic is in deepdoc/site/fumadocs_builder_v2.py. See Frontend Documentation Site Builder.

  • CLI Commands & Usage
    CLI entrypoints and command handling are in deepdoc/cli.py. See CLI Commands & Usage.

  • Integration Points
    LLM provider integrations (Anthropic, OpenAI, Azure, Ollama) are configured via deepdoc/config.py and used in deepdoc/llm/client.py. See Integration: Anthropic, Integration: Deepdoc Chat API, and Integration: Deepdoc Embed API.
